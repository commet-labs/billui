## Architecture

PaymentForm is **provider-agnostic**. It works with:

- **CardInput** — for demos, prototypes, or if you're building your own PSP
- **Stripe Elements** — iframes that handle PCI compliance
- **Adyen, Braintree, etc.** — any payment provider

`PaymentFormField` only renders **label + children + error/description**. Each child component handles its own styling.

### With Stripe Elements

Replace `CardInputGroup` with Stripe's `CardElement`. The layout stays the same.

```tsx title="stripe-checkout.tsx"
import { CardElement } from '@stripe/react-stripe-js';

function StripeCheckout() {
  const stripe = useStripe();
  const elements = useElements();

  const handleSubmit = async (e: React.FormEvent) => {
    e.preventDefault();
    if (!stripe || !elements) return;

    const card = elements.getElement(CardElement);
    const { token } = await stripe.createToken(card);
    // Send token to your server
  };

  return (
    <PaymentForm onSubmit={handleSubmit}>
      <PaymentFormSection>
        <PaymentFormField label="Card details">
          {/* Stripe Element renders an iframe here */}
          <div className="rounded-lg border bg-background p-3">
            <CardElement options={{
              style: {
                base: {
                  fontSize: '14px',
                  color: '#09090b',
                  '::placeholder': { color: '#a1a1aa' },
                },
              },
            }} />
          </div>
        </PaymentFormField>
      </PaymentFormSection>
      <PaymentFormFooter>
        <PaymentFormSubmit>Pay $99.00</PaymentFormSubmit>
      </PaymentFormFooter>
    </PaymentForm>
  );
}
```

<Callout type="info">
  **CardInput vs Stripe Elements**: CardInput is for demos and prototypes, or if you're building your own PSP and handle PCI compliance. For production payments, use Stripe Elements or similar—they handle sensitive card data in isolated iframes.
</Callout>

## API Reference

### PaymentForm

The root form element.

| Prop | Type | Default |
| --- | --- | --- |
| All `<form>` props | `FormHTMLAttributes` | — |

### PaymentFormSection

A container for grouping related fields.

| Prop | Type | Default |
| --- | --- | --- |
| `variant` | `"default" \| "elevated"` | `"default"` |

### PaymentFormHeader

Container for title and description.

### PaymentFormTitle

Section title (`<h3>`).

### PaymentFormDescription

Section description text.

### PaymentFormField

Wrapper for form fields with label and error handling. Only renders label + children + error/description—each child component handles its own styling.

| Prop | Type | Default |
| --- | --- | --- |
| `label` | `string` | — |
| `htmlFor` | `string` | auto-generated |
| `name` | `string` | — |
| `error` | `string` | — |
| `description` | `string` | — |
| `required` | `boolean` | `false` |

### PaymentFormRow

Grid container for side-by-side fields.

| Prop | Type | Default |
| --- | --- | --- |
| `columns` | `2 \| 3 \| 4` | `2` |

### PaymentFormFooter

Container for submit button and related actions.

### PaymentFormSubmit

Submit button with loading state and secure icon.

| Prop | Type | Default |
| --- | --- | --- |
| `loading` | `boolean` | `false` |
| `showSecureIcon` | `boolean` | `true` |
| All `<Button>` props | `ButtonProps` | — |

### PaymentFormDivider

Horizontal divider with optional label.

| Prop | Type | Default |
| --- | --- | --- |
| `label` | `string` | — |
